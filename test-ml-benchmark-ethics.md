# ML 基础知识：为什么基准测试、评估或测试集训练是大罪

机器学习不应该证明模型可以重复它已经见过的例子。它应该告诉我们该模型是否适用于下一个案例,即未在训练运行中的案例。这就是基准、评估和测试集存在的全部原因。

紧张之处在于:私人排名听起来可能像"相信我兄弟",但公布平均值可能会破坏使它们发挥作用的东西。一旦测试项目、评分逻辑、示例或答案模式成为公共优化目标,它们就不再是干净的测量,而是开始有可能成为基准测试实用程序。

基准、评估套件或隐藏测试很有用,因为它就像一个独立的测量仪器。它让我们可以问:"这个系统的泛化能力如何？"对该测量仪器的训练会破坏该仪器。它将分数变成曝光记录,而不是能力记录。

基本规则很简单:

> 使用培训数据进行培训。调整验证数据。老实说,根据测试数据进行一次测量。一旦您根据测试集进行训练、调整、提示优化、脚手架优化、过滤或选择,它就不再是测试集了。

几十年前,经典机器学习就理解了这一点。在拆分之前,您无法对完整数据集进行预处理。您不会使用测试标签选择功能。您不会通过查看最终测试性能来选择检查点。同样的逻辑也适用于 LLM,但表面积要大得多。

对基准、平均值或测试集进行训练并不是一个小的程序错误。这是一种大罪,因为它破坏了经验机器学习的核心对象:可信的泛化估计。

## ML 真正想做的事情

机器学习是一种从过去数据中学习的规范方法,因此系统在未来案例中表现良好。在监督学习中,通常的故事是输入、标签、模型、参数和损失函数。训练调整参数,以便预测能够改善训练示例。

但低训练损失并不是目标。目标是降低部署分布的预期损失。

这种区别就是一切。

由于多种原因,模型在训练数据上看起来不错:它可能学习了持久结构、记住了示例、找到了快捷方式、利用了数据集中的工件、从未来数据中发现了泄漏,或者学到了在基准上有效但在世界上失败的东西。

保留测试集的目的是将"该模型学习了任务"与"该模型学习了数据集"分开。

现代机器学习为数据提供了不同的功能:

- 训练集:适合参数、表示和学习到的特征。
- 验证或开发集:选择超参数、提示、架构、检查点。
- 测试集、评估或基准:估计决策冻结后的最终概括。
- 私人或隐藏测试:提供外部审计或排行榜分数。
- 部署监控数据:发布后跟踪现实世界的漂移和故障。

## 词汇:适合、概括、泄漏、污染

拟合意味着数据中的信息改变了系统。在经典 ML 中,拟合包括更新权重、估计归一化统计数据、选择特征、选择阈值、调整超参数和选择模型。

泛化意味着对目标分布中未见过的案例进行性能分析。测试集仅在独立于训练和选择过程的情况下估计泛化能力。

泄漏是指在训练、验证或评估过程中使用了预测时不应获得的信息。常见的泄漏模式包括预处理泄漏、特征泄漏、重复泄漏、群体泄漏、时间泄漏、目标泄漏、排行榜泄露和基准污染。

## 为什么法学硕士比传统机器学习更难污染

LLM 将文本转换为令牌 ID,通常使用 BPE、WordPiece 或 Unigram 等子词令牌化方案。Transformer 通过用注意力机制取代递归和卷积,使这种扩展变得实用。

现代法学硕士系统通常是分层构建的:预训练、领域预训练、监督微调、偏好优化、安全调整、工具使用、系统提示等。每个阶段都可能污染评估。

## 为什么测试集训练是一个基本罪过

1. 它破坏了测试集存在的理由
2. 它将科学变成自我欺骗
3. 它引发了一场基准黑客军备竞赛
4. 它打破了可比性
5. 它掩盖了现实世界的失败
6. 它削弱了安全评估

## 清洁基准实践

强有力的基准报告为读者提供了足够的细节来解释分数。至少报告:模型版本、发布日期、基础与微调状态、工具使用、RAG、净化方法、限制、提示、采样设置、评分代码、置信区间等。

一个严肃的基准主张应该能够说:

> "我们在接触测试集之前冻结了模型和评估协议。我们仅根据允许的培训数据进行培训。我们使用验证数据进行开发。我们检查了预训练、微调、偏好、合成和检索数据是否存在污染。"

## 最终原则

在基准、平均值或测试集上进行训练是一个大罪,因为它取代了我们真正关心的问题:"这个系统能处理新案件吗？"为"这个系统吸收了测量仪器吗？"

答案不是放弃基准。答案是尊重他们。使用培训数据进行培训。调整验证数据。对未触及的测试进行评估。对于重大索赔,请使用新鲜、私密、实时且经过充分审核的基准。

在机器学习中,测试集不仅仅是另一个数据集。它是统治者。一旦你弯曲尺子以适应模型,测量就消失了。
